# 專案設定與發佈

當所需的鏡像檔全數上傳完畢之後，接下來就是要將它打包成可發佈的版本。點選 Helm 頁籤進入版本發佈頁面。

# 新增 Helm

新增 Helm 文件。有兩種方式，分別是「Create New Helm」和「Import Helm」。

# 首次新增

若為首次新增 Helm 文件，可點擊 新增。

# 非首次新增

非首次新增 Helm 文件，可選擇建立新 Helm 檔，也可選擇修改指定版本的 Helm 檔。

# Helm 文件設定

文件設定的部份，依照選擇建立的 Helm 類型，有不同的項目需要分別設定。

詳細的操作與設定請參照以下兩個章節：

  • Container 設定
  • Virtual Machine 設定

# 驗證與發佈

  • 編輯的過程中，可以透過左側的 按鈕做設定上的確認

  • 編輯完成後，點選右下角的 按鈕來發佈

# 版本簽署狀態

在 Helm 設定完成且發佈後，該版本的簽署狀態會處於 Processing 狀態。待簽署完成後，狀態轉為 Published 則表示該版本正式發佈成功。

# 操作

在發布的版本列表中，可進行以下三種操作：

# 查看已發佈版本

選定任一版本並點擊 按鈕，可查看該版本的詳細設定。

詳細的設定說明，請參照以下兩個章節：

  • Container 設定
  • Virtual Machine 設定

# 下載

選定任一版本並點擊 按鈕，可下載該版本的壓縮檔。

# 設定預設版本

選定任一版本並點擊 ，將該版本設定為開發者預設版本。
