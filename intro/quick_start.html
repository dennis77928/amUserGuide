    # 快速入門

    本章節將協助您建立新專案，並發佈一個新的 Helm 文件。

    前提條件

    我們假設您已擁有管理者 (Admin) 或開發者 (Developer) 的帳號與密碼，並且已經登入 Application Manager 平台。

    # 1. 建立新專案

    開發者登入 AM 後，可看到畫面如下圖：

    • 點選 以新增專案

    • 為新專案命名

    • 選擇新建的專案後，填寫專案資訊

    # 2. 上傳 Image

    專案開啟後，SaaS 應用開發者便可透過 docker push 的指令將鏡像上傳至 AM 的專案內。

    # 指令格式

    • Tag
    docker tag SOURCE_IMAGE[:TAG] harbor.corepro.com.tw/{{**your_App_Name**}}/IMAGE[:TAG]
    • Push
    docker push harbor.corepro.com.tw/{{**your_App_Name**}}/IMAGE[:TAG]

    # 3. 專案設定

    # 新增 Helm 文件

    切換至 Helms 分頁，點擊

    點擊 ，選擇 Container 開始設定。

    # Container 設定

    • Overview 分頁

    在 Version 欄位填入發佈應用的版號。

    • Containers 分頁

    選定容器所使用的鏡像檔與標籤。

    # 4. 驗證與發佈

    • 點擊左側的 按鈕做設定上的驗證。

    • 驗證無誤後，點選右下角的 按鈕來發佈。

    # 5. 簽署狀態

    在 Helm 設定完成且發佈後，該版本的簽署狀態會處於 Processing 狀態。待簽署完成後，狀態轉為 Published 則表示該版本正式發佈成功。

    完成以上步驟，表示您已成功建立專案且發布一個新的 Helm 文件，詳細設定可參考 Container 設定 或 Virtual Machine 設定 章節的說明。
